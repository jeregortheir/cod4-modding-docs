# 🗺️ VLCT Deathrun - Руководство для Мапперов

Защитное руководство по скриптингу GSC для карт **CoD4 Deathrun**. Паттерны, рецепты и правила выживания, которые предотвращают краши сервера.

> **Новичок?** Начни с [Быстрого старта](#быстрый-старт-5-шагов) ниже, потом прочитай [Основы](/en/fundamentals.md) (на английском) - это всё что нужно для написания первой ловушки.

---

## Быстрый старт (5 шагов)

Самый быстрый путь от нуля до работающей карты:

1. **Скопируй файл шаблона**
   `maps\mp\_TEMPLATE_FOR_MAPPERS.gsc` → `maps\mp\mp_dr_ТВОЁИМЯ.gsc`

2. **Открой его.** Прокрути до `main()`. Удали строки `thread` для фич, которые тебе **не** нужны (например, удали `thread combat_room_sniper();` если на карте нет снайперской комнаты). Оставь `maps\mp\_load::main();` сверху.

3. **Для каждой фичи, которую оставил,** найди её секцию в этом руководстве и поправь `targetname` строки из Radiant (значения в `getEnt("...")`) чтобы соответствовать тому, что ты разместил в Radiant.

4. **Скомпилируй `.bsp`** в Radiant (`Compile > BSP + Light + Link`). Твой `.gsc` будет запечён в `.ff` при `linkMap`.

5. **Тест:** `/map mp_dr_ТВОЁИМЯ` в консоли сервера. После каждого теста проверь `qconsole.log` на `script runtime error`.

---

## Зачем это руководство

Движок GSC в CoD4 имеет острые края - одна незащищённая строка может:

* Положить **весь сервер** (например, вызов несуществующего билтина типа `setmovespeed()`)
* Спамить тысячи ошибок в минуту (например, доступ к `level.activ.X` когда никто не в команде активатора)
* Тихо сломать твою карту (например, дубликат `targetname` в Radiant → `getEnt` возвращает undefined)
